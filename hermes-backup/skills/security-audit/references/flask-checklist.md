# Flask Security Checklist

Framework-specific checks for Flask applications. Work through each section systematically.

## 1. Configuration Security

### SECRET_KEY
```python
# BAD: Hardcoded
app.config['SECRET_KEY'] = 'my-secret'

# GOOD: From environment with random fallback
app.config['SECRET_KEY'] = os.environ.get('SECRET_KEY', os.urandom(24).hex())
```
- **Check:** Is it from `os.environ`? Is the fallback truly random (not a fixed string)?
- **Pitfall:** `os.urandom(24).hex()` as a default means sessions break on restart if no env var is set. Document this.

### Debug Mode
```python
# BAD: Always on
app.run(debug=True)

# GOOD: Env-gated, off by default
app.run(debug=os.environ.get('FLASK_DEBUG', '0') == '1')
```
- **Check:** Is debug off by default? Is it only togglable via env var?
- **Why it matters:** Debug mode exposes the Werkzeug debugger, which allows arbitrary code execution.

### Database URI
- SQLite is fine for dev. Check that production uses PostgreSQL/MySQL with proper credentials.
- `SQLALCHEMY_TRACK_MODIFICATIONS = False` should be set (performance + deprecation warning).

## 2. CSRF Protection

### Flask-WTF Setup
```python
from flask_wtf.csrf import CSRFProtect
csrf = CSRFProtect()
csrf.init_app(app)
```

### Template Coverage
Every `<form method="post">` MUST have:
```html
<input type="hidden" name="csrf_token" value="{{ csrf_token() }}">
```

**Audit method:** Count `<form` tags with `method="post"` in templates. Count `csrf_token` hidden inputs. Numbers must match. Missing CSRF on any POST form = CRITICAL.

**Pitfall:** Forms generated by JavaScript (AJAX submissions) need CSRF tokens too. Check for `fetch()` or `axios.post()` calls that might skip CSRF.

## 3. XSS Prevention

### Jinja2 Auto-Escaping
Jinja2 auto-escapes `{{ var }}` by default. This protects **text content** and most **attribute values** (quoting `"→&quot;`).

### Where Auto-Escaping IS Safe
- `{{ variable }}` in text context
- `<input value="{{ variable }}">` (quotes are escaped)
- `{{ variable }}` inside `href`, `src` (quotes are escaped)

### Where Auto-Escaping Is NOT Sufficient
1. **`| safe` filter** — disables escaping entirely. FIND AND ELIMINATE unless the content is pre-sanitized.
2. **`{% autoescape false %}`** — disables escaping for a block.
3. **`Markup()` in Python** — marks string as safe, bypasses escaping.

### The `| e` vs `| safe` Decision
```jinja
{{ article.content | safe }}   # DANGEROUS: renders raw HTML, XSS vector
{{ article.content | e }}      # SAFE: escapes HTML, shows tags as text
```

**Trade-off:** `| e` breaks legitimate HTML rendering (articles with `<p>`, `<a>` tags show raw source). The secure fix is:
1. Sanitize input with `bleach` before storage (whitelist safe tags)
2. Use `| safe` in template (content is already sanitized)

### Avatar/User-Generated URLs
```html
<img src="{{ user.avatar_url }}">
```
Auto-escaping prevents attribute breakout, but validate `avatar_url` server-side:
- Must start with `https://`
- Reject `javascript:` and `data:` URIs
- Reject URLs with newlines or control characters

## 4. Authentication & Authorization

### Login Redirect (`next` parameter)
```python
# DANGEROUS: Open redirect
next_page = request.args.get('next')
return redirect(next_page or url_for('main.index'))

# SAFE: Validate the URL
from urllib.parse import urlparse
next_page = request.args.get('next')
if next_page:
    parsed = urlparse(next_page)
    if parsed.netloc or not next_page.startswith('/'):
        next_page = None
return redirect(next_page or url_for('main.index'))
```

### Logout CSRF
```python
# VULNERABLE: GET allows CSRF logout via <img src="/auth/logout">
@auth_bp.route('/logout')
def logout(): ...

# BETTER: POST-only with CSRF token
@auth_bp.route('/logout', methods=['POST'])
@login_required
def logout(): ...
```

### Password Policy
- Minimum 8 characters (ideally 12+)
- Check server-side, not just HTML5 `required` attribute
- Consider: `werkzeug.security.generate_password_hash` uses pbkdf2/scrypt by default (good)

### Registration Validation
- Username: length limits, character whitelist, uniqueness check
- Email: format validation (server-side regex), uniqueness check
- Password: minimum length, complexity if needed
- **Pitfall:** Check both username AND email uniqueness. Unhandled `IntegrityError` from DB constraint = 500 error.

### Admin Authorization
```python
def admin_required(f):
    @wraps(f)
    @login_required
    def decorated(*args, **kwargs):
        if not current_user.is_admin:
            flash('دسترسی غیرمجاز', 'danger')
            return redirect(url_for('main.index'))
        return f(*args, **kwargs)
    return decorated
```
- Verify EVERY admin route uses `@admin_required` (not just `@login_required`)
- Check that `is_admin` is not user-settable (no form field, not in registration)

## 5. Session Security

```python
app.config['SESSION_COOKIE_SECURE'] = True     # HTTPS only
app.config['SESSION_COOKIE_HTTPONLY'] = True    # No JS access (Flask default)
app.config['SESSION_COOKIE_SAMESITE'] = 'Lax'  # CSRF mitigation
```

- `httponly`: Flask defaults to True — verify it's not overridden
- `secure`: Set True in production (requires HTTPS)
- `samesite`: 'Lax' or 'Strict' — prevents CSRF on cross-origin GET

## 6. Security Headers

Add via `@app.after_request`:

```python
@app.after_request
def security_headers(response):
    response.headers['X-Content-Type-Options'] = 'nosniff'
    response.headers['X-Frame-Options'] = 'DENY'
    response.headers['X-XSS-Protection'] = '1; mode=block'
    response.headers['Content-Security-Policy'] = (
        "default-src 'self'; "
        "img-src 'self' https: data:; "
        "style-src 'self' 'unsafe-inline' https://fonts.googleapis.com https://cdn.jsdelivr.net; "
        "script-src 'self' https://cdn.jsdelivr.net; "
        "font-src 'self' https://fonts.gstatic.com"
    )
    return response
```

- `X-Content-Type-Options: nosniff` — prevents MIME sniffing
- `X-Frame-Options: DENY` — prevents clickjacking
- `Content-Security-Policy` — controls allowed resources. Customize per project.

## 7. Rate Limiting

```python
from flask_limiter import Limiter
from flask_limiter.util import get_remote_address

limiter = Limiter(get_remote_address, app=app)

@auth_bp.route('/login', methods=['GET', 'POST'])
@limiter.limit("5/minute")
def login(): ...

@auth_bp.route('/register', methods=['GET', 'POST'])
@limiter.limit("3/minute")
def register(): ...
```

**Minimum rate limits:**
- Login: 5/minute per IP
- Registration: 3/minute per IP
- Password reset: 3/hour per email
- API endpoints: 60/minute per user

## 8. Input Validation

### Server-Side Validation
HTML5 `required`, `type="email"`, `maxlength` are client-side only. Always validate server-side.

### File Upload (if present)
- Validate file type by content (magic bytes), not extension
- Store outside webroot
- Rename uploaded files (don't use user-provided filename)
- Limit file size

### SQLAlchemy Query Safety
```python
# SAFE: Parameterized query
User.query.filter_by(username=username).first()

# DANGEROUS: String interpolation (SQL injection)
db.session.execute(f"SELECT * FROM user WHERE username = '{username}'")
```
- SQLAlchemy ORM queries are parameterized by default — safe
- Raw SQL via `db.session.execute()` needs parameter binding
- `.contains()`, `.filter()` with `==` are safe; `.filter(User.name == var)` is safe

## 9. Error Handling

```python
# BAD: Leaks internal info
flash(f'Error: {str(e)}', 'danger')

# GOOD: Generic message, log the detail
import logging
logger = logging.getLogger(__name__)
logger.error(f"Operation failed: {e}", exc_info=True)
flash('خطا در پردازش. لطفاً دوباره تلاش کنید.', 'danger')
```

- Never show raw exception messages to users (even admins)
- Use `logging` for debugging, generic messages for UI
- `exc_info=True` logs the full traceback server-side

## 10. Seed / Development Data

- Seed scripts often contain hardcoded passwords (`admin123`)
- Verify seed scripts are never run against production
- If seed data includes admin accounts, generate random passwords and print them once
- Check for `.env.example` with placeholder values (not real secrets)

## 11. Dependency Audit

Check `requirements.txt` / `Pipfile` for:
- Known CVEs (use `pip-audit` or `safety check`)
- Pinned versions (not `>=` ranges)
- Unnecessary dependencies (attack surface)
- `bleach` for HTML sanitization (if rendering user HTML)

```bash
pip-audit -r requirements.txt
```

## Common Flask Vulnerabilities Quick Reference

| Vulnerability | Where to Check | Severity |
|--------------|---------------|----------|
| Hardcoded SECRET_KEY | `__init__.py`, `config.py` | CRITICAL |
| debug=True in production | `run.py`, `app.py` | CRITICAL |
| Missing CSRF on POST forms | All `templates/*.html` | CRITICAL |
| XSS via `\| safe` | All `templates/*.html` | CRITICAL |
| Open redirect via `next` | Auth login route | CRITICAL |
| No rate limiting on auth | Auth routes | HIGH |
| No password policy | Registration route | HIGH |
| Logout via GET | Auth logout route | HIGH |
| Missing security headers | `@app.after_request` | MEDIUM |
| No session cookie flags | `__init__.py` config | MEDIUM |
| Hardcoded seed passwords | `seed.py` | MEDIUM |
| Exception messages exposed | Error handlers, flash() | LOW |
| No input validation | Registration, profile edit | HIGH |
