# Pattern: Data Persistence

## تشخیص سریع — چه بپرسی تا بفهمی Data Persistence هست؟
- [ ] کاربر گفت "ذخیره / دیتابیس / جدول / شیت"؟
- [ ] "دیتا رو نگه داره / بعداً بتونم ببینم / تاریخچه"؟

## سوالات هدفمند (قبل از جستجو)
| بپرس | تا بفهمی |
|------|---------|
| چقدر دیتا ذخیره میشه؟ (چند ردیف / روزانه) | حجم و مقیاس |
| کاربر میخواد دیتا رو ببینه و ویرایش کنه؟ | → Sheets یا Data Table |
| رابطه بین جداول داره؟ | → Airtable یا Postgres |
| فقط اضافه کنه یا آپدیت هم کنه؟ | Append vs Update |
| چن تا فیلد داره؟ اسمشون چیه؟ | Schema |

## کلمات کلیدی برای جستجوی مشابه
`data storage`, `n8n google sheets`, `database automation`, `log data`

## Credentialهایی که باید چک کنی
Google Sheets / Airtable / Postgres / MySQL / MongoDB
