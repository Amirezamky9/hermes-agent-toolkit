# E-Commerce Platform Build — Real Execution Benchmark (2026-07-13)

## Project Goal
Build a full-stack e-commerce platform (React + FastAPI + SQLite) with:
- User auth (JWT + bcrypt)
- 50 products across 5 categories
- Shopping cart + checkout flow
- Order management + reviews
- Admin panel + seed data
- Docker setup + documentation

## Execution Method
3 parallel Hermes subagents via `delegate_task()`:
- **Subagent 1**: Backend (11 Python files) — 252s, 45 API calls
- **Subagent 2**: Frontend (1 HTML React SPA) — 145s, 11 API calls  
- **Subagent 3**: Supporting (seed data, tests, Docker, README) — 267s, 37 API calls

### Bugs Found During Build
1. **JWT `sub` field type** — `user.id` is int, python-jose requires string
   - Fix: `str(user.id)` in token creation, `int(payload.get("sub"))` in decode
   - Impact: All authenticated endpoints return 401
   - This is a COMMON pattern across FastAPI projects generated by subagents

2. **None** — all other endpoints worked first-time

### Architecture Lessons for v3.6
1. **Seed data is essential** — 50 products + 3 users + orders + reviews in one script
2. **Mock data fallback** — frontend should work standalone without API
3. **Docker compose** — backend (uvicorn) + frontend (nginx) need separate services
4. **JWT `sub` must be string** — recurring subagent bug, should be a verifier check
5. **Agent skills per role** — BACKEND_DEV needs FastAPI + SQLAlchemy + JWT knowledge
6. **Research phase would help** — auth pattern research before coding avoids the JWT bug
7. **No type hints** — subagents don't add type hints; need explicit instruction
